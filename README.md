# Omok

### 소개
GitHub의 협업 기능으로 실제 팀 프로젝트가 진행되는 과정을 체험하고 

현재 학습 중인 Java를 복습하기 위해 Java를 활용한 콘솔 기반 오목 게임을 기획했습니다.


### 목적
- Java의 기존에 알던 내용과 원하는 기능을 구현하기 위해 더 알아야할 내용이 있는지 탐색하고

    더 알아봄으로써 배우는 과목에 대한 심화학습
- GitHub 에서 각각의 브랜치를 만들고 그 브랜치 안에서 하나의 프로젝트를 위해 코드를 짜고

    Pull Request를 위해 상대의 코드를 점검하며 협업을 체험

### 아쉬운 점
- 구현된 오목을 플레이하려면 반드시 2명 이상의 사람이 요구됩니다. 개인적인 욕심으로 혼자 인공지능과 상대하는 기능을 구현해 보고 싶었지만 아무곳에나 돌을 랜덤하게 놓는 기능이 아닌 이기려는 목적을 가지고 더 승률이 높은 곳을 계산해 돌을 놓는 기능은 오목 실력의 부족함과 코드의 구현력 부족으로 실현되지 못해 아쉽다고 생각합니다.
- 한번도 협업을 진행해 본적이 없는 상태에서 진행된 과제인 만큼 처음 느껴보는 불편함이 몇 가지 있었습니다. 개개인이 기능을 구현할 때 생각하는 방식이나 코드를 구현하는 방식이 다른데 이를 그냥 작성하고 넘기니 확인하고 다음 작업을 수행할 때 팀원이 어떤 생각과 의도를 가지고 코드를 작성하거나 수정했는지 파악하는 것이 꽤 번거롭다고 느껴졌습니다. 그래서 프로젝트의 중간부터는 Pull Request의 설명에도 어떤 코드를 몇 쨰줄에 추가했는지나 코드의 몇몇 내용을 주석에 추가하는 방식으로 불편함을 덜어내려고 했으나 초반에 그런 불편함을 예상하지 못하고 진행된 것이 아쉬웠습니다.
